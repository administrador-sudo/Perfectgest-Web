Papéis no tratamento (LGPD)
• Controlador dos dados clínicos dos pacientes: a clínica ou o profissional de saúde que utiliza o software (define finalidades e meios do tratamento no contexto da assistência).
• Operador do software: Perfect Gest Dev (PERFECT GEST DESENVOLVIMENTO DE SOFTWARE LTDA) — desenvolve e mantém a ferramenta; não decide sobre o conteúdo clínico inserido pela clínica.
• Direitos do titular (dados de saúde): dirigir-se primeiro à clínica (controlador). Suporte técnico do produto: suporte@perfectgestdev.com.

1. Escopo

O PerfectGest-Clinica III é um sistema de gestão clínica para Windows. O app armazena dados clínicos localmente no dispositivo (contentor MSIX). A clínica ou profissional que utiliza o software é responsável pela inserção e gestão dos dados.

2. Dados tratados

O app trata dados inseridos pelo utilizador (ex.: pacientes, prontuário, agenda, equipa). Por padrão, não enviamos conteúdo clínico para a nuvem.

2.1 Dados locais (por padrão)
• Pacientes e prontuário (saúde) — SQLite local
• Agenda e registos de operação — SQLite local
• Sessão/conta local e perfis (ADMIN/DOUTOR/OPERADOR) — SQLite local

2.2 Dados que podem sair do dispositivo (opcionais / por configuração)
• Pagamentos de subscrição: Microsoft Store (Microsoft Corporation). A Perfect Gest Dev não armazena dados de cartão.
• WhatsApp Cloud API: se configurado pela clínica, mensagens são enviadas à Meta (WhatsApp).
• Firebase (opcional): licenças remotas e configurações técnicas (sem prontuário).
• Relatório técnico de instabilidade (com consentimento): relatório anónimo, sem dados de paciente.

3. Finalidades

• Operar as funcionalidades do app (gestão clínica e organizacional)
• Suporte técnico quando solicitado
• Cumprimento de obrigações legais/regulatórias

4. Base legal (LGPD)

O tratamento de dados de saúde ocorre no contexto da clínica (controlador). O software oferece mecanismos de segurança e auditoria para apoiar a conformidade.

5. Segurança

• Auditoria de acesso: metadados (sem conteúdo clínico no log), retenção configurável
• Backup encriptado (ZIP): AES-256 + manifesto HMAC
• Encriptação da base de dados (opcional): SQLite + passphrase no Credential Locker do Windows

6. Retenção e eliminação

• Dados no dispositivo enquanto o app estiver instalado
• Desinstalação remove dados do contentor MSIX; backups exportados ficam sob responsabilidade da clínica

7. Direitos do titular

• Acesso/correção via ficha do paciente
• Relatório LGPD (ADMIN) em PDF/CSV
• Portabilidade via backup encriptado
• Pedidos adicionais: contactar a clínica (controlador) ou suporte@perfectgestdev.com (suporte técnico)

8. Contato

suporte@perfectgestdev.com

Roles in processing (LGPD)
• Controller of patients' clinical data: the clinic or health professional using the software (defines purposes and means of processing in the care context).
• Software operator: Perfect Gest Dev (PERFECT GEST DESENVOLVIMENTO DE SOFTWARE LTDA) — develops and maintains the tool; does not decide on clinical content entered by the clinic.
• Data subject rights (health data): contact the clinic first (controller). Product technical support: suporte@perfectgestdev.com.

1. Scope

PerfectGest-Clinica III is a clinical management system for Windows. The app stores clinical data locally on the device (MSIX container). The clinic or health professional using the software is responsible for entering and managing data.

2. Data processed

The app processes data entered by the user (e.g. patients, medical records, schedule, team). By default, we do not send clinical content to the cloud.

2.1 Local data (by default)
• Patients and medical records (health) — local SQLite
• Schedule and operation logs — local SQLite
• Local session/account and roles (ADMIN/DOCTOR/OPERATOR) — local SQLite

2.2 Data that may leave the device (optional / by configuration)
• Subscription payments: Microsoft Store (Microsoft Corporation). Perfect Gest Dev does not store card data.
• WhatsApp Cloud API: if configured by the clinic, messages are sent to Meta (WhatsApp).
• Firebase (optional): remote licenses and technical settings (no medical records).
• Technical crash report (with consent): anonymous report with no patient data.

3. Purposes

• Operate app features (clinical and organizational management)
• Technical support when requested
• Compliance with legal/regulatory obligations

4. Legal basis (LGPD)

Health data processing occurs in the clinic's context (controller). The software provides security and audit mechanisms to support compliance.

5. Security

• Access audit: metadata (no clinical content in logs), configurable retention
• Encrypted backup (ZIP): AES-256 + HMAC manifest
• Database encryption (optional): SQLite + passphrase in Windows Credential Locker

6. Retention and deletion

• Data remains on the device while the app is installed
• Uninstalling removes data from the MSIX container; exported backups remain the clinic's responsibility

7. Data subject rights

• Access/correction via patient record
• LGPD report (ADMIN) in PDF/CSV
• Portability via encrypted backup
• Additional requests: contact the clinic (controller) or suporte@perfectgestdev.com (technical support)
